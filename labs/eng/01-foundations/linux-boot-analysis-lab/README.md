Week 1 Lab — Linux System Boot Analysis
1. Objective

The objective of this lab is to understand the different stages of the Linux system boot process:

Firmware (BIOS / UEFI)

Bootloader (GRUB)

Kernel loading

Service initialization (systemd)

User session start

5. Linux Kernel Verification

Command

uname -r


Meaning

uname displays system information

-r shows the Linux kernel version

Observed result

5.4.0-84-generic


Observation

The Linux kernel loaded during the boot process is version 5.4.0-84-generic.

6. Kernel Message Analysis

Command

dmesg | head


Result

[    0.000000] Linux version 5.4.0-84-generic
[    0.000000] Command line: BOOT_IMAGE=/boot/vmlinuz
[    0.000000] BIOS-provided physical RAM map
[    0.000000] NX (Execute Disable) protection: active


Observation

These messages are generated by the Linux kernel during system startup.

They indicate:

kernel loading

memory detection

hardware initialization

memory protection mechanisms (NX)

7. Boot Log Analysis

Command

journalctl -b | head


Result

Mar 14 10:22:14 ubuntu kernel: Linux version 5.4.0-84-generic
Mar 14 10:22:14 ubuntu systemd[1]: Started Journal Service.
Mar 14 10:22:15 ubuntu systemd[1]: Started Network Service.
Mar 14 10:22:15 ubuntu systemd[1]: Reached target Network.


Observation

This command displays logs from the last system boot.

It allows us to:

view boot events

identify services started during startup

detect potential errors

8. Identification of the First Process

Command

ps -p 1


Result

PID TTY          TIME CMD
1 ?        00:00:01 systemd


Observation

The process with PID 1 is systemd.

Role of systemd

starts system services

organizes the boot process

supervises running services

9. Analysis of Active Services

Command

Result

networking.service        loaded active running Network Manager

Observation

These services are automatically started by systemd to provide essential system functionality.

10. Global Boot Process Overview
Stage	Function
Power	supplies electricity and starts the CPU
Firmware	initializes hardware components
Bootloader	loads the Linux kernel
Kernel	initializes system resources
systemd	starts and manages system services
Services	provide system functionalities
User Session	allows the user to access the system
11. Possible Issues

Some problems may prevent the system from booting correctly:

Corrupted bootloader → the kernel cannot be loaded

Corrupted kernel → system cannot start

systemd missing or blocked → services cannot start
